> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Hold the bid on a profitable, non-defense keyword once it has reached its top-of-search target (`target_tos_share ≥ tos_target_pct`) instead of raising it further, and remove the three superseded/declined TOS bid-up code paths.

**Architecture:** All changes are in one file — `scripts/bigquery/views/V_ADS_COACH.sql` (the live coacher decision view). Add one `WHEN` to the `target_action` CASE (the brake), delete the inert TOS `INCREASE_BID` branch from the same CASE, delete the TOS size-up branch from both the `recommended_bid` and `bid_change_pct` CASEs, and reword the `tos` decision-trace chip. There is no SQL unit-test harness in this repo; each task is validated by deploying the view (`CREATE OR REPLACE VIEW` via `bq query`) and running read-only `bq` validation queries, the same pattern used for sub-projects A and B.

**Tech Stack:** BigQuery Standard SQL; `bq` CLI (authed for `onyga-482313`); the view is `onyga-482313.OI.V_ADS_COACH`.

- [ ] **Step 1: Insert the brake branch immediately before the scale-up frequency-gate tier**


Insert this brake branch **directly above** the `-- ═══ FREQUENCY GATE` comment, so the brake is evaluated before the scale-up/profitable INCREASE tiers but after the peak re-increase and bleeder blocks already above it:

```sql
    -- ═══ TOS BRAKE: already-dominant profitable keyword → hold the bid ═══
    -- Once target_tos_share ≥ tos_target_pct the term owns its top-of-search position;
    -- raising the bid further buys CPC, not impressions. Hold (no change). Defense exempt
    -- (the brand moat always defends). Self-correcting: if TOS later erodes below target,
    -- this WHEN fails and the normal scale-up/profitable raise resumes. (Coacher A.1.)
    WHEN d.target_tos_share IS NOT NULL
         AND d.tos_target_pct IS NOT NULL
         AND d.target_tos_share >= d.tos_target_pct
         AND d.target_roas >= d.th_profitable_roas AND d.eff_orders_for_bid >= 2
         AND d.strategy_id NOT IN ('BRAND_DEFENSE', 'PRODUCT_DEFENSE')
      THEN 'MONITOR_TARGET'

```
