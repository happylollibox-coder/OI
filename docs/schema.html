BRIDGE FACT_FACTLESS_BRIDGE (Central Hub)

FACT_FACTLESS_BRIDGE ▼

Factless fact table linking all (date_key, asin) combinations. Populated by SP_POPULATE_FACTLESS_BRIDGE. Enables unified analysis across facts.

ColumnTypeNotes
date_keyINT64YYYYMMDD, -1 if missing
asinSTRINGUNKNOWN if missing
factless_keySTRINGdate_key-asin (e.g. 20260201-B09XQ56RK5)

Facts contributing to bridge (via UNION in SP_POPULATE_FACTLESS_BRIDGE):

  • ✓ FACT_INVENTORY_SNAPSHOT
  • ✓ FACT_FINANCIAL_TRANSACTIONS
  • ✓ FACT_PURCHASE_ORDER
  • ✓ FACT_AMAZON_PERFORMANCE_DAILY (factless_key = YYYYMMDD-PURCHASED_ASIN)
  • ✓ FACT_AMAZON_SEARCH_PERFORMANCE_WEEKLY (factless_key = Reporting_Date-ASIN)
  • ✓ FACT_EXPERIMENT_DAILY (factless_key = snapshot_date-ASIN)

Facts NOT in bridge: FACT_AMAZON_ADS (uses Ads_key, campaign/ad_group/keyword grain), FACT_SEARCH_QUERY, FACT_ORDERS

Relationships

Join Paths (Bridge-Centric) ▼

Via FACT_FACTLESS_BRIDGE: Facts join on factless_key. Bridge links to DIM_TIME (date_key) and DIM_PRODUCT (asin).

  • → FACT_FACTLESS_BRIDGE.factless_key = FACT_AMAZON_PERFORMANCE_DAILY.factless_key
  • → FACT_FACTLESS_BRIDGE.factless_key = FACT_AMAZON_SEARCH_PERFORMANCE_WEEKLY.factless_key
  • → FACT_FACTLESS_BRIDGE.factless_key = FACT_INVENTORY_SNAPSHOT.factless_key
  • → FACT_FACTLESS_BRIDGE.factless_key = FACT_PURCHASE_ORDER.factless_key
  • → FACT_FACTLESS_BRIDGE.factless_key = FACT_EXPERIMENT_DAILY.factless_key
  • → FACT_FACTLESS_BRIDGE.date_key = DIM_TIME.date_key
  • → FACT_FACTLESS_BRIDGE.asin = DIM_PRODUCT.asin

Experiment & Ads (not via bridge):

  • → DIM_EXPERIMENT 1:* DIM_EXPERIMENT_CAMPAIGN
  • → DIM_EXPERIMENT_CAMPAIGN.campaign_id = FACT_AMAZON_ADS.campaign_id
  • → FACT_AMAZON_ADS.most_advertised_asin_* = DIM_PRODUCT.asin
  • → FACT_AMAZON_ADS + V_SRC_AmazonAds_campaign_history (date BETWEEN OI_start_date, OI_end_date)
  • → campaign_history.portfolio_id = V_SRC_AmazonAds_portfolio.portfolio_id
  • → FACT_EXPERIMENT_DAILY.experiment_id = DIM_EXPERIMENT.experiment_id
  • → FACT_EXPERIMENT_DAILY.asin = DIM_PRODUCT.asin

DIM_COSTS_HISTORY (qualified join): Full key is (marketplace_id, asin, sku). Views use p.asin = ch.asin with ROW_NUMBER() OVER (PARTITION BY asin ORDER BY end_date DESC) = 1 for latest cost. Procedures filter marketplace_id = 'ATVPDKIKX0DER'. Simple asin-only join is a simplification.

  • → DIM_PRODUCT.asin = DIM_COSTS_HISTORY.asin (+ temporal: latest row per asin)
  • → DIM_PRODUCT.asin = V_SRC_sales_and_traffic.child_asin

Tested: factless_key referential integrity (FACT_AMAZON_SEARCH_PERFORMANCE_WEEKLY, FACT_AMAZON_PERFORMANCE_DAILY). See scripts/bigquery/tests/
